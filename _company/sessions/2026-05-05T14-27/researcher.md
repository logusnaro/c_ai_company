## 행동-결과 인과관계 학습을 위한 최소 행동-결과 매핑 데이터 유형 및 수집 우선순위 정의

CEO의 지시와 회사 목표(AI 인사이트 기반 행동 변화 유도)에 따라, 행동(Action)과 결과(Result)의 인과관계를 학습하는 데 필요한 최소한의 데이터 유형(Feature Set)과 수집 우선순위를 정의합니다.

### 1. 행동-결과 인과관계 학습을 위한 최소 Feature Set 정의

AI가 '왜(Why)' 특정 행동이 특정 결과를 낳았는지 학습하기 위해서는 단순히 기록(Log)하는 것을 넘어, 행동과 결과 사이의 **맥락(Context)**과 **주관적 평가(Self-Reflection)**를 명시적으로 연결해야 합니다.

| 엔티티 | 핵심 필드 (Feature Set) | 설명 및 인과관계 기여도 | 행동 변화 연관성 |
| :--- | :--- | :--- | :--- |
| **Action (행동)** | `action_id`, `action_type`, `timestamp`, `raw_data` | 사용자가 수행한 구체적인 행위. (독립 변수, 원인) | 무엇을 했는지 기록 |
| **Context (맥락)** | `location`, `mood_state`, `external_factors` | 행동이 발생한 환경 및 심리적 상태. (행동의 조건) | 행동에 영향을 미친 외부 요인 파악 |
| **Result (결과)** | `outcome_metric` (정량적), `self_reflection` (주관적) | 행동의 직접적인 결과 및 사용자의 주관적 평가. (종속 변수, 결과) | 행동의 성공/실패 여부 측정 |
| **Causality Link (인과 연결)** | `causal_hypothesis` (선택 사항), `time_lag` (선택 사항) | AI가 추론할 수 있도록 행동과 결과 간의 잠재적 인과관계에 대한 초기 가설 또는 시간 지연 정보를 제공. | AI의 추론 깊이 향상 |
| **System State (시스템 상태)** | `input_friction_score` (자동 측정), `data_completeness` | 데이터 입력 과정에서의 마찰도 및 데이터의 완전성. (행동을 유도한 시스템의 영향) | 사용자의 행동 변화 유도 방식 분석 |

### 2. 데이터 수집 우선순위 재확인 (MVP 기준)

회사 정체성("입력 마찰 최소화", "행동 변화 유도")과 단기 목표(MVP 출시)를 고려할 때, 데이터 수집의 우선순위는 **측정 용이성**과 **행동 변화에 대한 직접적인 연관성**에 중점을 두어야 합니다.

| 우선순위 | 데이터 유형 | 수집 목표 및 이유 | 관련 가치 |
| :--- | :--- | :--- | :--- |
| **🥇 1순위 (필수)** | **Action + Result (행동 및 결과)** | 사용자가 특정 행동을 취한 후, 그 결과로 측정 가능한 지표(Outcome Metric)가 발생했는지 기록하는 것. **(인과관계 학습의 기본)** | AI 인사이트의 근간 |
| **🥈 2순위 (핵심)** | **Context (맥락)** | 행동이 발생한 환경(장소, 기분)을 기록하여, 동일한 행동이 다른 맥락에서 어떻게 다른 결과를 낳는지 분석할 수 있게 한다. | 행동 변화 유도 (맥락 기반 코칭) |
| **🥉 3순위 (최적화)** | **System State (시스템 상태)** | 데이터 입력 시 사용자가 느낀 마찰도(Input Friction)나 데이터 입력의 완전성 정도를 기록한다. **(입력 마찰 최소화 목표 달성)** | 제품 개선 및 사용성 향상 |
| **선택 사항 (고급)** | **Causality Link (인과 연결)** | AI가 추론할 수 있도록 행동과 결과 사이의 잠재적 연결 고리나 시간적 지연 정보를 추가 수집한다. | AI 예측 및 코칭의 정교화 |
