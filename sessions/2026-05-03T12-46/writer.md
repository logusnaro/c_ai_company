# 👶 육아앱 MVP 기능 정의 초안 (AI 인사이트 중심)

본 문서는 '기록'을 넘어 '행동 변화'를 유도하는 AI 인사이트를 핵심 가치로 삼아, MVP에 포함되어야 할 기능 목록과 각 기능의 철학적 정의(Why)를 제시합니다.

---

## 💡 핵심 가치 제안 (Core Value Proposition)

**"당신의 육아 기록을 AI가 이해하고, 가장 효과적인 다음 행동을 제안하여, 당신이 원하는 부모가 되도록 돕는 지능형 파트너."**

---

## 📋 MVP 기능 목록 및 철학적 정의

| 기능 분류 | 기능 상세 (What) | 철학적 정의 (Why) | 우선순위 (MoSCoW) |
| :--- | :--- | :--- | :--- |
| **1. 데이터 입력 (Foundation)** | **음성 입력 자동 기록 시스템** | **'귀찮음'의 제거:** 부모가 복잡한 입력 과정 없이 자연스럽게 기록하게 하여, 기록 자체의 허들을 0으로 만든다. (Pain Point 해결) | Must-have |
| | **최소 데이터 입력 및 저장** | **'기록'의 시작:** 필수적인 데이터만 요구하여, 사용자가 앱에 머무르는 시간을 최소화하고 기록 루틴을 시작하게 한다. | Must-have |
| **2. AI 인사이트 (Core Value)** | **AI 기반 일일/주간 요약 리포트** | **'해석'의 자동화:** 방대한 기록을 단순한 숫자가 아닌, 부모의 행동 변화에 직접 연결된 '이야기'로 변환하여, 사용자가 데이터를 분석하는 데 드는 인지적 노력을 제거한다. | Must-have |
| | **단일 행동 제안 (Single Action Suggestion)** | **'행동 변화' 유도:** 단순한 분석을 넘어, '무엇을 해야 할지'에 대한 구체적이고 실행 가능한 다음 단계를 제시하여, 기록을 실제 행동으로 연결시킨다. (AI의 핵심 가치) | Must-have |
| **3. 습관 형성 (Engagement)** | **기록 루틴 알림 및 동기 부여** | **'지속성' 확보:** 기록을 일회성이 아닌 습관으로 만들기 위해, 심리학적 동기 부여 메커니즘을 적용하여 사용자가 꾸준히 앱을 사용하도록 유도한다. | Should-have |
| | **기록 패턴 시각화 대시보드** | **'자기 인식' 증진:** 복잡한 데이터를 시각적 패턴으로 제공하여, 사용자가 자신의 육아 패턴을 객관적으로 인식하고 개선할 수 있는 명확한 시각적 근거를 제공한다. | Should-have |
| **4. 커뮤니티 (Expansion)** | **안전한 공유 기능 (선택)** | **'연결'의 확장:** 신뢰하는 사람들과 기록의 일부를 공유함으로써, 고립된 경험을 집단 지성으로 확장하고 정서적 지지를 얻을 수 있는 통로를 제공한다. | Could-have |

## ✍️ Writer 개인 메모리 (자기 점검)

- **핵심 확인:** MVP는 '기록 앱'이 아니라 **'행동 변화 파트너'**여야 한다. 모든 기능은 궁극적으로 사용자가 '더 나은 부모'가 되는 행동으로 이어져야 한다.
- **톤앤매너:** 직설적이고 친근하게, 복잡한 AI 개념을 사용자의 일상적인 고민 해결에 직접 연결하여 설명해야 한다.
- **다음 스텝 준비:** 이 정의를 바탕으로 후크와 핵심 메시지를 작성하여 시장에 내놓을 준비를 한다.
