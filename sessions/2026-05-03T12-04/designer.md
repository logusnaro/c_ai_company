## 1. 전체 사용자 흐름 (Flow)

**목표:** 사용자가 생각하지 않고 말하면, 자동으로 기록되도록 만드는 것. (빠르고 자연스러운 기록 경험 최우선)

**사용자 흐름 (Happy Path):**

1.  **시작 (Initiation):** 사용자가 앱에서 음성 입력 버튼(🎤)을 탭한다.
2.  **입력 (Input):** 시스템이 음성 녹음을 시작하고 사용자에게 "듣고 있어요" 상태를 표시한다.
3.  **STT & NLP (Processing):** 음성이 텍스트로 변환(STT)되고, 텍스트가 의도(Intent) 및 구조화된 데이터로 분석된다.
4.  **확인 요청 (Validation):** 시스템은 분석된 의도를 기반으로 구조화된 데이터 초안을 생성하고, 사용자에게 **구체적인 확인 질문**을 제시한다. (예: "수유 120ml 기록할까요?")
5.  **사용자 응답 (Confirmation):** 사용자는 [확인] 또는 [수정] 버튼 중 하나를 선택한다.
6.  **최종 저장 (Final Save):** 사용자가 확인을 승인하면, 데이터는 최종적으로 DB에 구조화되어 저장된다.
7.  **피드백 (Feedback):** 성공적인 기록 완료 메시지를 사용자에게 제공한다.

**예외 흐름 (Error/Ambiguity Path):**

1.  **STT 실패:** 음성 인식이 불가능할 경우 → "다시 말씀해 주세요" 메시지 출력.
2.  **의미 불명확 (Low Confidence):** NLP 모델이 의도를 확신하지 못할 경우 → 시스템이 추측 대신 **확인 요청 단계**로 즉시 진입한다. (예: "무슨 기록인지 잘 모르겠어요. 수유 기록인가요?")
3.  **데이터 누락:** 숫자나 필수 정보가 누락된 경우 → "수유 양을 알려주세요"와 같이 **보완 요청**을 한다.

## 2. 상태 정의 (State)

시스템은 다음의 핵심 상태들을 관리해야 한다.

| 상태 ID | 상태 명칭 | 설명 | 주요 액션 |
| :--- | :--- | :--- | :--- |
| **S1** | **대기 (Idle)** | 앱이 기본 상태로 대기하며, 음성 입력 버튼을 기다리는 상태. | 🎤 탭 대기 |
| **S2** | **녹음 중 (Recording)** | 사용자가 음성을 입력하는 동안 시스템이 STT를 처리하는 상태. | 음성 입력 감지 |
| **S3** | **분석 중 (Analyzing)** | STT 결과가 텍스트로 변환된 후, NLP/LLM이 의도 및 구조를 분석하는 상태. | (시스템 내부 처리) |
| **S4** | **확인 요청 (Confirming)** | 시스템이 분석 결과를 바탕으로 사용자에게 확인 질문을 제시하는 상태. | [확인] / [수정] 선택 대기 |
| **S5** | **저장 중 (Saving)** | 최종 확인이 완료된 데이터가 DB에 쓰여지는 상태. | (시스템 내부 처리) |
| **S6** | **완료 (Success)** | 모든 기록 과정이 성공적으로 완료된 상태. | 기록 완료 알림 |
| **S7** | **오류 (Error)** | STT 실패, 의미 불명확 등 처리 중 오류가 발생한 상태. | 재시도 또는 오류 메시지 표시 |

## 3. 에러 케이스 (Error Cases)

| 에러 유형 | 발생 조건 | 시스템 대응 전략 | 사용자 피드백 예시 |
| :--- | :--- | :--- | :--- |
| **STT 실패** | 음성 인식이 불가능하거나 잡음이 심할 때. | 즉시 녹음을 중단하고 재시도 요청. | "음성 인식이 되지 않았습니다. 다시 말씀해 주세요." |
| **의미 불명확 (Low Confidence)** | 입력된 텍스트의 의도나 키워드가 모호할 때. | 즉시 S4 (확인 요청) 상태로 전환. | "혹시 수유 기록을 말씀하시는 건가요? 수유 기록을 확인하시겠어요?" |
| **데이터 불완전** | 수유량이나 시간 등 필수 숫자가 누락되었을 때. | S4에서 필수 항목 보완 요청. | "수유 양(ml)을 알려주세요." |
| **처리 시간 초과** | LLM/NLP 처리 시간이 목표치(1초)를 초과할 때. | 사용자에게 잠시 기다려 달라고 안내 후 재시도 유도. | "잠시만 기다려 주세요. 분석 중입니다." |
| **권한 오류** | (잠재적) 외부 API 연동 중 권한 문제 발생 시. | 사용자에게 권한 재설정을 요청. | "기록을 위해 필요한 권한이 부족합니다." |

## 4. UX 화면 정의 (Screen Definition)

**A. 음성 입력 중 화면 (S2)**

*   **레이아웃:** 전체 화면을 음성 입력 상태에 집중.
*   **상단:** 타이머 또는 진행 표시 (선택 사항).
*   **중앙:** **'듣고 있어요' 상태 표시.** (시각적 피드백)
*   **하단:** **가이드 텍스트** 표시.
    *   *예시 1 (초기):* "기록하고 싶은 내용을 말씀해 주세요."
    *   *예시 2 (진행 중):* "음성을 분석하여 기록을 정리하고 있습니다..."
