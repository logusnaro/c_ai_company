# 📋 작업 브리프

**원 명령:** 지금 기획 중인 육아앱 관련해서 앞서 말한 것들이야.
사장이 도망갔는데 잡아왔나 모르겠네. 사장은 아래거 읽고 현재 기획안에 반영할 수 있도록 해.
나머지는 하던 일 계속해요.

1. 폰트 등 디자인적 요소들은 지적재산권에 어긋나지만 않으면 괜찮아.
2. 최대한 무료로 하고 싶고, 안티그래비티에서 구현할거니까 거기에 맞춰서 짜면 됨.
3. 입력방식은 텍스트와 음성 두가지로 할거야. 음성은 자연어처리로 사용자가 대충 말해도 이해하여 자동 저장하게 했으면 좋겠어
4. 그리고 AI로 DB를 분석해서 제공하겠다는 서비스가 있었는데 그럼 AI와 질의응답하는 기능도 있어야하니까 참고해
5. 아래는 예전에 gpt랑 대화한 내용으로 만든 프롬프트인데, 참고해서 쓸만한 것들은 쓰고 참고 안해도 되는 내용들은 버려버려.

[역할]
너는 모바일 앱 프로덕트 디자이너 + 시니어 엔지니어다.
초보 부모를 위한 "음성 기반 육아 기록 기능"을 설계하고 구현한다.

이 기능의 목표는:
"사용자가 생각하지 않고 말하면, 자동으로 기록되도록 만드는 것"이다.

---

[핵심 컨셉]

1. 입력은 단순해야 한다 (1~2초)
2. 사용자는 판단하지 않는다 (자동 해석)
3. 시스템은 확신이 없으면 반드시 확인을 요청한다
4. 절대 복잡한 입력 UI를 만들지 않는다
5. 정확도보다 "사용 흐름의 자연스러움"이 더 중요하다

---

[기능 정의]

사용자는 하단 중앙의 🎤 버튼을 눌러 음성 입력을 시작한다.

시스템은 다음 흐름으로 동작해야 한다:

1. 음성을 텍스트로 변환 (STT)
2. 텍스트를 분석하여 의도를 파악
3. 구조화된 데이터로 변환
4. 사용자에게 확인 요청
5. 확인 시 저장

---

[지원 기능 범위 - MVP]

다음 4가지 기록만 지원한다:

1. 수유 (feeding)
2. 수면 (sleep)
3. 기저귀 (diaper)
4. 신체 (body)

---

[의도 해석 기준]

다음과 같은 자연어 입력을 처리해야 한다:

- "120 먹였어" → 수유
- "잠들었어" → 수면 시작
- "깼어" → 수면 종료
- "끙가했어" → 대변
- "쉬했어" → 소변

---

[데이터 처리 원칙]

- 모든 입력은 구조화된 데이터로 변환한다
- 시간은 자동 기록한다
- 숫자는 자동 추출한다
- 불확실한 경우 "unknown" 처리한다

---

[중요: UX 흐름]

절대 바로 저장하지 말고,
항상 아래 형태의 확인 단계를 거쳐야 한다:

"수유 120ml 기록할까요?"

[확인] [수정]

---

[예외 처리]

다음 상황을 반드시 처리해야 한다:

1. 의미를 이해할 수 없는 경우
→ "무슨 기록인지 잘 모르겠어요" 메시지

2. 애매한 경우
→ confidence 낮게 설정 후 확인 유도

3. 숫자가 없는 수유 입력
→ 사용자에게 보완 요청

---

[성능 최적화 전략]

- 단순 키워드/패턴은 코드로 처리
- 복잡한 문장만 LLM 사용
- 응답 속도는 1초 이내 목표

---

[UI 요구사항]

음성 입력 중:
- "듣고 있어요" 상태 표시
- 간단한 가이드 문장 제공

결과 화면:
- 한 줄 요약
- 큰 버튼 2개 (확인 / 수정)

---

[금지 사항]

- 자유 입력 폼 만들지 마라
- 복잡한 설정 추가하지 마라
- 설명형 UI 만들지 마라
- 여러 단계 입력 흐름 만들지 마라

---

[출력 요구]

이 기능을 다음 형태로 설계하라:

1. 전체 사용자 흐름 (Flow)
2. 상태 정의 (State)
3. 에러 케이스
4. UX 화면 정의
5. 데이터 처리 방식

---

이 기능은 MVP이며,
"빠르고 자연스러운 기록 경험"이 최우선이다.

## 요약
모바일 앱의 음성 기반 육아 기록 기능 MVP를 위한 전체 사용자 흐름, 상태 정의, UX 화면, 데이터 처리 방식을 설계합니다.
